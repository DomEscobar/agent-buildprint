# Design Quality Bar

Build a dense, operational simulation workbench. Avoid marketing hero layouts, decorative cards, static screenshots as primary content, and one-note palette choices.

The graph and runtime surfaces are primary product surfaces and must be legible under real data volume. Controls must be stable in size, toolbars must use familiar icons where available, destructive controls must be visually distinct, and status text must not overlap in compact panels.

Accessibility gates: keyboard access for workflow controls, visible focus states, readable contrast, non-color-only status indicators, responsive layout without clipped labels, and screen-readable error/progress messages.
