# Design Quality Bar

Build a dense, operational simulation workbench. Avoid marketing hero layouts, decorative cards, static screenshots as primary content, and one-note palette choices.

The graph and runtime surfaces are primary product surfaces and must be legible under real data volume. Controls must be stable in size, toolbars must use familiar icons where available, destructive controls must be visually distinct, and status text must not overlap in compact panels.

Accessibility gates: keyboard access for workflow controls, visible focus states, readable contrast, non-color-only status indicators, responsive layout without clipped labels, and screen-readable error/progress messages.


## Product Category

Serious operator workbench for source-independent AI workflow execution.

## Taste Direction

Precise, calm, data-dense, and production-grade; avoid generic demo-app styling.

## Visual Hierarchy

Primary workflow status, current artifact, blockers, and next action must be immediately scannable.

## Forbidden Generic Patterns

No placeholder dashboards, lorem ipsum cards, disconnected mock graphs, static success shells, or decorative-only progress UI.

## Interaction Polish

Controls expose pending/disabled/retry states and preserve context during async updates.

## Accessibility Gates

Visible focus, keyboard path, labelled controls, and contrast-safe status treatment are blocking.

## Responsive Gates

Empty: screenshot required. Loading: screenshot required. Error: screenshot required. Blocked: screenshot required. Success/ready: screenshot required.

## Required Screenshot Set

Empty: required. Loading: required. Error: required. Blocked: required. Success/ready: required.
