# Agent Handoff

Use this late. `BUILDPRINT.md`, `PLAN.md`, `questions.md`, and the numbered phase files are higher authority.

## Mission Reminder

Map a source repository into a scoped, evidence-backed Buildprint package. The mapper must discover first, ask later, avoid source mutation, avoid secret values, and prefer smaller complete scope over broad fake coverage.

## Latest Safe Next Step

- If safety/read/export boundaries are unclear: finish `plans/00-safety-boundaries.md`.
- If repo facts are thin or filename-only: continue `plans/01-repo-census.md`.
- If architecture zones lack evidence labels: continue `plans/02-system-map.md`.
- If candidates are vague or merged: redo `plans/03-candidate-buildprints.md`.
- If no candidate/scope/fidelity is confirmed: continue `plans/04-scope-decision.md`.
- If selected package lacks execution rails, QA, traceability, or no-fake gates: redo `plans/05-single-extraction.md`.
- If hierarchical/system extraction was requested: continue `plans/06-system-extraction.md`.
- If tests, reversal, golden evals, QA evidence, or final handover are missing: continue `plans/07-validation-submission.md`.
- If all gates pass: no next step is needed unless the user asks for publishing, website sync, or deeper parity proof.

## Final Chat Handover

State outcome, selected scope, evidence inspected, files generated, commands/evals run, known gaps, and recommended next direction.
