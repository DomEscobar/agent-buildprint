# Final Reviewer

Review the artifact before you explain it. Use the artifact type from `blueprint.yaml`: product app, framework, library, integration, automation, data pipeline, infrastructure, or mixed.

1. Start fresh and complete the core loop for the real consumer.
2. Reload/restart/rerun and verify meaningful state, traces, or outputs return where the artifact requires continuity.
3. Change inputs/config/events and verify output/artifacts/behavior change.
4. Click visible primary controls or run documented commands/API calls/operator actions.
5. Trigger empty, error, blocked-provider, and reset/delete states where possible.
6. Look for generic dashboard smell, fake intelligence, raw JSON dumped as the experience, placeholder copy, dead controls, undocumented public methods, fake adapter seams, canned output, internal/proof vocabulary, missing persistence, and absent next actions.
7. Fix local, safe, central defects before handover.

Leave only real blockers: credentials, destructive actions, deployment, security review, paid services, environment failure, or large scope.
