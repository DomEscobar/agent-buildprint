# Phase 01 — First usable product loop

## Product intention

Create the smallest end-to-end path that lets a real user experience the product promise.

## Build

- The main input path.
- The central artifact or work surface.
- One meaningful interaction with that artifact.
- The first useful output/result.
- Local persistence or readback where the product needs continuity.
- Honest blocked states for unavailable live providers or external services.

## Quality bar

A user should understand what to do, see that their input changed the product, and know the next action without reading the packet.

## Do not ship

Decorative artifact, generic dashboard, raw JSON output, dead controls, canned results, or fake live-provider behavior.
