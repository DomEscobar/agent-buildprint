# Phase Flow

For each phase:

1. Read the phase and restate the product intention in one paragraph.
2. Build the smallest real usable slice for the declared artifact type.
3. Improve the obvious next consumer action if it is local, safe, and central: user action, developer adoption step, operator recovery, task approval, dataflow inspection, or infrastructure operation.
4. Run the strongest relevant local check.
5. Remove visible slop before moving on.
6. Record only useful handover facts: works, partial, blocked, next repair.

Do not create long self-assessments. Do not treat a passed command as artifact quality. Do not move on while the main visible control, documented command, public method, adapter seam, operator action, or recovery path in the phase is fake, dead, or disconnected from real state.
