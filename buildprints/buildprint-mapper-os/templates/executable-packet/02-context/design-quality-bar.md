## Forbidden Generic Patterns

- Generic dashboard card grid as the main app.
- Decorative landing-page hero for an operational workspace.
- Raw JSON textarea as the main product surface.
- One-note color palette with no hierarchy.
- Buttons or tabs that do not execute real behavior.

## Interaction Polish

- Every visible control must have a real action, disabled reason, or explicit blocker.
- Long operations must show progress or pending state.
- Error states must explain recovery.
- Export/report actions must produce a file, preview, or blocker artifact.

## Accessibility Gates

- Keyboard reachable primary flow.
- Visible focus styles.
- Form controls have accessible labels.
- Color is not the only status signal.

## Responsive Gates

- Desktop proof required.
- Mobile or narrow viewport proof required.
- Text must not clip, overlap, or force layout jumps.
- Toolbars and status pills must wrap predictably.
