# UX Contract

## Screen Inventory

- Ingest form.
- Ingest result state.
- Error state.

## Workflow States

- Empty:
- Loading:
- Error:
- Blocked:
- Success/ready:

## Component Inventory

- Form controls.
- Submit action.
- Result list.
- Error panel.

## Responsive Rules

- Desktop and mobile layouts must not clip text or controls.

## Accessibility

- Keyboard path, labels, focus visibility, contrast, and error messaging are required.

## Browser Proof Plan

- Screenshot or browser trace must cover empty, loading, error, blocked, and success/ready states or record an explicit blocker.

## Non-Negotiable UI Failures

- Generic dashboard card grid.
- Static markup with dead buttons.
- No browser proof.
