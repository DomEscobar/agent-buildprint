# Agent Prompt

You are building a trusted-local MiroFish-style swarm prediction workbench. This prompt is alignment speech, not authority; use `BUILDPRINT.md` and `blueprint.yaml` for the contract.

Build the product loop a user can actually feel: seed material goes in, a graph comes out, the canvas makes the graph inspectable, simulation setup can use that graph, and reports/interactions do not pretend provider work happened when credentials are missing.

The graph canvas is the center of gravity. Preserve zoom, selection, labels, node and edge details, self-loop and multi-edge readability, refresh, maximize, and empty/loading/error states.

Replace Zep Cloud properly. A rename will fail because the source behavior used Zep for graph construction, ontology application, episode ingestion, readback, search, stats, report tools, and optional live memory updates. Build a neutral graph-memory port and use an OSS backend behind it.

Keep AI provider choice open. Use OpenAI-compatible settings as the default integration surface, but never hard-code a vendor into product behavior. Missing credentials are a blocked live proof, not an excuse for canned output.

Avoid generic dashboards, raw JSON product surfaces, fake intelligence, canned reports, dead buttons, placeholder copy, swallowed errors, and proof vocabulary in the user/operator experience.
