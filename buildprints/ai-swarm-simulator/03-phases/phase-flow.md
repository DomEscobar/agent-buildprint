# Phase Flow

For each phase:

1. Restate the product intention in one paragraph.
2. Build the smallest real slice that moves the user through the prediction workbench.
3. Keep the graph-memory port, provider boundaries, persistence, and canvas experience honest.
4. Improve the obvious next user action when it is local, safe, and central.
5. Run relevant checks and record exact failures.
6. Remove visible slop: dead controls, placeholder copy, raw JSON surfaces, fake progress, fake intelligence, and provider errors hidden as success.
7. Update handover with only useful facts: commands run, surfaces built, blockers, next atomic action.
