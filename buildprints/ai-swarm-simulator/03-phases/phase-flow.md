# Phase Flow

For each phase:

1. Read the phase file and restate the product intention in your own words.
2. Build the smallest real usable slice for this artifact type.
3. Improve the obvious next consumer action if it is local, safe, and central.
4. Run relevant checks: command, API call, browser flow, restart/readback, or provider blocked-state check.
5. Remove visible slop: dead controls, fake progress, raw JSON surfaces, swallowed errors, confusing empty states, placeholder text, and generic dashboard panels.
6. Record only useful handover facts: what works, what was checked, what is blocked, and the next atomic action.

If a live provider is missing, implement the boundary honestly. The user should see which provider is blocked, what credential or sandbox is needed, and what can still be inspected locally.

If a phase reveals a product contradiction, update the packet-local handover notes and repair the central loop before adding breadth.
