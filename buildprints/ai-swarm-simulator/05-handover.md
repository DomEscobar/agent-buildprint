# Handover

Use these short headings in the final implementation handover.

## Current status

State whether this is local build, provider-blocked build, sandbox-verified build, or another honest status. Do not claim production-ready or complete unless that was actually achieved and reviewed.

## Built surfaces

List the user-facing and operator/developer surfaces built: upload/project, graph, simulation prepare, run timeline, report, interaction, setup/deploy.

## Verification

List commands, API calls, browser flows, reload/restart checks, and provider checks actually run. Include blocked provider checks as blocked, not skipped success.

## Known defects and blockers

Name remaining issues plainly: credentials, sandbox budget, auth/deployment, persistence limitations, OASIS runtime gaps, destructive-action policy, UI gaps.

## Next atomic actions

List the next concrete actions in order. Prefer one-step items such as "add durable task store", "run open-source graph memory sandbox graph build", or "add auth gate before public deploy".
