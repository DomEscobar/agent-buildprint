# UX UI Craft Contract

## When Active

Use this role for UI-bearing phases, especially Canvas, workbench, visual node states, responsive behavior, and screenshot review.

## Handoff Scope

- Protect the product-grade Canvas experience.
- Define interaction polish, responsive layout, accessibility, and visual proof expectations.
- Review screenshots and browser flows, not just component code.

## Reject If

- The primary UI is generic forms, raw tables, unstyled lists, default controls, or static screenshots.
- Canvas nodes lack media previews, status, spatial relationships, inspector affordances, or clear hierarchy.
- Text overlaps or controls resize unpredictably.
- Visual quality relies on a one-note palette, placeholder graphics, or local-MVP screenshots.
- Mobile/tablet layout is unusable.

## Required Return Headings

- Canvas Experience Review
- Visual Quality Findings
- Interaction Findings
- Accessibility/Responsive Findings
- Screenshot Proof Needed
- Required Repairs

## Proof/Evidence Expectations

Require desktop and mobile screenshots, browser e2e traces for key interactions, and explicit pass/fail notes for empty, loading, populated, failed, and blocked states.
