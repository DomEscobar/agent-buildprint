# Phase 00 - Alignment

## Goal

Configure the fixed portable storyboard proof without redesigning it.

## Keep In Context

- `BUILDPRINT.md`
- `questions.md`
- `PRODUCT_QUALITY_BAR.md`
- `PARITY_CLAIMS.md`

## Steps

1. If the user said `Use default storyboard preset`, continue with the default values in `questions.md`.
2. Otherwise ask exactly `questions.md`.
3. Summarize choices and wait for confirmation.
4. Start `VALIDATION.md` from `VALIDATION_TEMPLATE.md` and record alignment choices.
5. Record target shape constants from `BUILDPRINT.md` in the implementation report.

## Do Not

- ask broad product strategy questions;
- add live provider, Electron, exact canvas/UI, or final video export parity;
- let alignment become a redesign phase;
- skip product-quality target recording.

## Exit Criteria

- default preset accepted or confirmed alignment choices exist;
- provider mode is mock/no-network by default;
- product-quality target is recorded as creative workbench first and debug evidence secondary.

## Validation Evidence

- `VALIDATION.md` alignment choices section filled;
- implementation report confirms binding slice and non-goals.
