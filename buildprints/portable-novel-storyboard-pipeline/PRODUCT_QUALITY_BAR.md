# Product Quality Bar

## Attention Anchor

Reread before UI implementation:

- Creative workbench first.
- Debug evidence secondary.
- Mock/no-network default.
- Preview manifest only.
- No original UI/provider/Electron/final video parity claims.

## Minimum Product Standard

The browser result must feel like a professional creative storyboard workbench. Passing data-contract tests is not enough.

Primary user experience:

- Storyboard frames or deterministic local thumbnails are visible in the main preview.
- Timeline lanes show ordered clips, duration, track grouping, and media state.
- Selecting a shot opens an inspector with frame, video description, prompt, linked assets, duration, and task state.
- Asset and media state use compact tiles, badges, and thumbnails.
- Task log, raw manifest JSON, raw mock/provider refs, and validation internals live in a debug drawer or details view.
- Limitations/non-claims are visible near export/settings, but they must not dominate the creative workspace.

## Forbidden Product Outcomes

Do not accept these as product-grade:

- Raw media URI table as the primary media UI.
- Manifest textarea dominating the preview page.
- Empty mobile dashboard screenshot used as responsive proof.
- Task log as the main product surface.
- Generic admin dashboard/card soup counted as a creative workbench.
- Raw `mock://...` refs wrapping inside narrow table cells.
- Placeholder controls counted as implemented behavior.

## Screenshot Bar

Acceptance screenshots must show completed product state:

- Desktop: timeline lanes, storyboard frames/thumbnails, selected-shot inspector, compact media/task state, and export affordance.
- Mobile: completed preview or storyboard state, not only the initial project dashboard.
- No overlapping critical controls, unreadable text, or label splitting such as `ima/ge`, `vid/eo`, or broken status words.
