# Questions

## Required Before Reversal

| # | Question | Safe default |
|---|---|---|
| 1 | Target implementation stack? | TypeScript service skeleton with mocks |
| 2 | Export format? | JSON manifest plus zip bundle |
| 3 | Include minimal UI? | No UI for Phase 1; service/API proof only |
| 4 | Fixture style/story pack? | One art style plus one story style |

## Non-Blocking

- Should live provider adapters be stubbed only, or should one provider have a real integration test behind env vars?
- Should auth be included in the proof or mocked with a local project token?
