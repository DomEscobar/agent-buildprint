# questions

Ask only what is needed for the chosen fidelity target.

1. Target stack: Node/React, Python/FastAPI/Vue, or other?
2. UI target: WebUI only, CLI only, or both?
3. Providers: OpenAI-compatible, Anthropic, Bedrock, Ollama/local, or fake-only?
4. Tool policy: should shell/filesystem/network tools exist, and what approval model is required?
5. Memory persistence: in-memory, local files, SQLite, Postgres?
6. Skills: folder-based local skills, marketplace/imported skills, or hardcoded built-ins?
7. MCP: which servers/tools should be enabled, if any?
8. Subagents: simple delegated tasks or real parallel workers?
9. Browser QA: should the final implementation include a real UI runtime test?
10. What parity target is desired beyond the default mocked runtime proof?
