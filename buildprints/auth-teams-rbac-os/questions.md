# Configuration Questions

Ask only questions that block safe implementation.

1. Should the product call the container team, org, workspace, company, or something else?
2. Can a user belong to multiple teams?
3. Can users create their own teams after signup?
4. Is there a personal/default workspace?
5. Are roles fixed or custom?
6. Can invites be accepted only by exact email, or by domain policy?
7. Who may manage billing?
8. Can admins invite owners, or only owners?
9. Can an owner leave when another active owner exists?
10. Should deleted users remain visible in audit logs?
11. Are API keys team-scoped, user-scoped, or both?
12. Does the current app have legacy solo-user records needing backfill?
