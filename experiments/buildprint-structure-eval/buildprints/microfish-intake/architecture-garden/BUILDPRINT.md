# Architecture-Garden Buildprint — Microfish Intake Slice

## Operating rule

Build the Microfish slice while preserving future product shape. Every feature slice must include a bounded architecture/UX garden pass: name the seam, keep it testable, and avoid local-MVP collapse that blocks the next slice.

## Required read order

1. `BUILDPRINT.md`
2. `SYSTEM_SHAPE.md`
3. `FEATURE_SLICE.md`
4. `GARDEN_BUDGET.md`
5. `VERIFY_AND_REVIEW.md`
6. `HANDOVER.md`

## Execution loop

```text
understand system shape -> build feature slice -> garden architecture/UX -> verify -> handover
```

Do not spend unlimited time polishing architecture. The garden budget is bounded and must serve the product slice.
