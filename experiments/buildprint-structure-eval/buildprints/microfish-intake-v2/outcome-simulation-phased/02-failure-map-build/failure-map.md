# Failure Map

| id | Failure | Product response |
|---|---|---|
| F-01 | User does not know what to do first. | One clear headline, one primary CTA, example notes. |
| F-02 | User sees technical jargon before value. | Plain-language labels; advanced details collapsed. |
| F-03 | Result is a data dump. | Summary story: what we found, why it matters, next step. |
| F-04 | Input changes do not matter. | Evidence-sensitive analysis and test. |
| F-05 | User loses work. | Local save/readback. |
| F-06 | User cannot trust provider/runtime claims. | Gentle “runs locally” explanation; live providers hidden in details. |
| F-07 | Code becomes a throwaway demo. | Modular TS architecture and tests. |
