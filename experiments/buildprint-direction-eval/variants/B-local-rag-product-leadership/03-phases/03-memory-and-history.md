# Phase 03 — Memory and History

## Product intention

The user can return to prior work and preserve useful notes without confusing memory with source truth.

## Build

- Chat/session history.
- Pinned memory/note area if useful.
- Restore prior question/answer with citations.
- Distinguish user notes from source documents.
- Reload persistence.

## Quality bar

A user should know what came from documents, what they wrote, and what the system inferred locally.

## Do not ship

Memory that silently changes answers without visibility, history that disappears, or notes masquerading as source facts.
