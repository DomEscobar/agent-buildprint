# Phase 01 — Document Library and Ingestion

## Product intention

The user can add local knowledge and immediately see what was indexed.

## Build

- Paste and/or upload text/markdown documents.
- Document list with title, size, chunk count, updated time.
- Chunk preview or source drawer.
- Empty, validation, and ingestion states.
- Local persistence.

## Quality bar

The user should trust what entered the system. If a format is unsupported, say so clearly.

## Do not ship

Upload button that does nothing, fake indexed counts, raw JSON library, or hidden corpus state.
