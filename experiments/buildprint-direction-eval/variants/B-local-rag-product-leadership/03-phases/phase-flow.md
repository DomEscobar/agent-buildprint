# Phase Flow

For each phase:

1. Read the phase and restate the product intention in one paragraph.
2. Build the smallest real usable slice.
3. Improve the obvious skeptical-user check if it is local, safe, and central.
4. Run the strongest relevant local check.
5. Remove visible slop before moving on.
6. Record only useful handover facts: works, partial, blocked, next repair.

Do not create long self-assessments. Do not treat a passed command as product quality. Do not move on while the visible control for the phase is fake.
