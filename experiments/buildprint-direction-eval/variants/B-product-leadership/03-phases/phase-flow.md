# Phase Flow

For each phase:

1. Read the phase and restate the product intention in one paragraph.
2. Build the smallest real usable slice.
3. Improve the obvious next user action if it is local, safe, and central.
4. Run the strongest relevant check.
5. Remove visible slop before moving on.
6. Record only what matters in handover: works, partial, blocked, next repair.

Do not create long self-assessments. Do not treat a passed command as product quality. Do not move on while the main visible control in the phase is dead or fake.
