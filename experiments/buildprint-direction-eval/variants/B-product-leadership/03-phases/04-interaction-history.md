# Phase 04 — Interaction and History

## Product intention

The user can ask follow-up questions or make adjustments against the current story artifact, and the product keeps useful history.

## Build

- Interaction console or adjustment panel tied to the current artifact.
- Deterministic/local response path.
- History list with meaningful labels and timestamps.
- Ability to revisit prior generated artifacts or notes.

## Quality bar

Interaction must use current project context. It should not feel like a generic chatbot bolted onto the side.

## Do not ship

Chat box that ignores story state, fake assistant persona, history that disappears on reload, generic lorem output.
