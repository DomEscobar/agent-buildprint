# Phase 01 — Intake and Workbench

## Product intention

The user can create or open a story project, add seed material, and immediately see a meaningful workbench artifact generated from it.

## Build

- Project/session creation.
- Seed input via paste and/or sample seed.
- A central workbench artifact: graph, scene map, or storyboard board.
- Selection/inspection for at least one node/scene/entity.
- Empty, loading, and error states.
- Local persistence for the seed and artifact.

## Quality bar

The artifact must not be decorative. Selection should reveal useful details and relationships. Changing seed input should change the artifact.

## Do not ship

Static SVG, token bubbles, raw JSON as product, generic dashboard cards, dead inspector.
